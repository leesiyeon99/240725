심화과제1.StringBuilder 클래스
게임에서 string 형이 자주 변하면 생기는 문제를 서술하고, StringBuilder는 어떠한 장점이 있는지도 조사하여 해당 내용을 기입하시오. StringBuilder의 경우, 게임의 어느 상황에서 사용되면 유리할 지 기입하시오.

String은 불변성의 특징을 가지고 있다. 즉 잘 변하지 않는다는 것인데 변하지 않는다는 것은 해당 sting변수에 어떤 값이 들어간 다름 다른 값이 추가되거나 들어있는 값을 삭제하는 등 값의 추가/삭제가 이루어지지 않는다는 것이다.
string a라는 변수에 값을 담기 위해 hello 메모리를 할당한다면 "hello" 값이 담긴 메모리가 생성되고 이 메모리 주소를 가리키는 string 변수 a가 만들어진다.
여기서 world라는 문자열을 더해주면 기존의 생성된 메모리에 값이 추가되는 것이 아닌 helloworld라는 값을 가진 새로운 string을 만들고 이에 맞는 새로운 메모리를 할당 받게 된다 그러면 기존의 hello의메모리 공간은 자동적으로 쓰레기가 된다 만약 문자열을 계속 변경하거나 추가한다면 많은 메모리를 잡아먹고 속도가 줄어들 수 있다.

그래서 이 문제를 해결하기 위해 stringbuilder를 사용한다.
가변성의 특징을 가지고 있어 문자열이 추가삭제될 때 내부적으로 문자열을 수정하고 재할당 하는 대신 내부 버퍼에서 작업을 수행한다. 이는 문자열을 반복적으로 수정할 때 메모리 할당과 성능을 개선할 수 있다.

게임에서는 플레이어의 체력이 감소할 때마다 ui에 플레이어의 현재 체력을 표시하는 텍스트를 예로 들 수 있다. 또한 게임 내에서는 다양한 캐릭터가 존재한다고 하면 각각의 캐릭터의 속성을 설명할 떄 문자열을 동적으로 생성할 수 있을 것이다.

심화과제2.record
C#의 record는 C# 9.0부터 도입된 개념으로, 불변의 멤버 변수를 가질 수 있도록 하는 키워드입니다. C#의 record에 대해 조사해보고, 사용 예시를 들어 보고서를 작성하시오.

record는 불변 데이터 구조를 정의하기 위한 새로운 타입니다. 일반적인 객체와 차이점으로는 불변성을 강조한다는 것이다 이것은 record인스턴스가 한번 생성되면 내부 데이터를 수정할 수 없다 이러한 불변성은 코드를 예측 가능하고 오류를 줄이는데 도움된다. 또한 record는 값 의미 체계를 가진다 즉, record는 내용이 동일한 두 인스턴스가 실제로 같은 값을 나타내는 것으로 간주된다 이것은 객체의 비교를 쉽게 만들어준다.

만약 반에 같은 이름을 가진 친구가 있는지 찾는다고 할 때 record를 사용하지 않으면 같은 이름이더라도 저장되는 메모리가 다르기 때문에 false라고 나올 것이다 하지만 record는 값 취급되므로 이름이 같으면 같다라고 나온다.

1. record를 사용한다면 데이터 클래스를 정의하는데 많은 코드를 줄 일 수 있다. c#컴파일러가 자동으로 equals(), GetHashCode(), ToString() 메서드를 생성해준다.
2. 불변성이 보장된다 
3. 패턴 매칭을 사용할 때 유용하게 사용할 수 있다.
